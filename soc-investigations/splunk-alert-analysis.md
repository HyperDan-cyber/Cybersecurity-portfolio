# SIEM Alert Investigation: Suspicious Login Activity

## Project Overview

This project simulates a Security Operations Center (SOC) investigation using SIEM concepts.

The objective is to understand how security analysts review alerts, analyze event data, identify suspicious behaviour, and document response actions.

---

# Environment

Type:

Training / Simulated SOC Exercise

Technology:

SIEM concepts (Splunk fundamentals)

Purpose:

To practice security alert triage and incident investigation workflows.

---

# Scenario

A SIEM platform generates an alert indicating multiple failed login attempts against a user account.

The analyst is required to investigate whether the activity represents normal behaviour or a possible security threat.

---

# Investigation Workflow

## 1. Alert Review

The analyst reviews:

- Alert category
- Timestamp
- User account involved
- Source IP address
- Number of failed attempts

---

## 2. Event Analysis

The investigation focuses on:

- Authentication patterns
- Repeated failures
- Unusual login behaviour
- Possible brute-force indicators

---

# Findings

The activity may indicate:

- Incorrect password attempts
- User account targeting
- Automated login attempts

Additional investigation is required before confirming malicious activity.

---

# Recommended Actions

- Review successful login events after failures
- Verify user activity
- Check source IP reputation
- Enable MFA where appropriate
- Apply account protection policies

---

# SOC Analyst Skills Demonstrated

- Alert triage
- Security event analysis
- Incident documentation
- Threat detection concepts
- SIEM workflow understanding

---
